1. MCP 工具扩展（缺失 Resources/Prompts）

  - 当前只有 6 个工具，没有注册 MCP Resources（让 AI 直接浏览经验列表）
  - README 明确说"未注册 Resources / Prompts"
  - 建议：添加 mcp.resource 暴露经验列表、项目目录等，让 AI 不用工具调用就能拉上下文
  - 成本：低，API 已存在

  2. 搜索质量提升

  - intent_router 目前是 Default 实现，query 增强策略较简单
  - min_similarity 硬编码（solve=0.5, search=0.6, suggest=0.4）
  - 建议：
    - 引入 learning-to-rank 或更智能的 query 扩展
    - 相似度阈值自适应（根据 query 长度、类型动态调整）
    - 增强 include_archives 的混合搜索精度

  3. Personal Memory 画像质量

  - _try_extract_and_save_personal_memory 的 LLM 解析超时仅 25s
  - build_profile_for_user 的 max_per_side 默认 20 条，可能不够
  - 建议：增加解析重试机制、支持更多 preference 类型

  4. Hermes Daemon 与主服务解耦

  - 最近 commit 大量涉及 TM Daemon (scripts/daemon/)
  - 当前 daemon 通过 HTTP API forward 到主服务
  - 建议：独立部署能力、daemon 自身监控指标

  中等成本优化

  5. Legacy 搜索回退简化

  - search_orchestrator.py 有完整的 _legacy_search fallback
  - 说明新 pipeline 还有边界 case 没覆盖
  - 建议：补齐 pipeline 能力，逐步废弃 legacy

  6. Janitor 自动化的完善

  - janitor_scheduler 存在但功能较简单
  - 建议：
    - 自动合并重复经验
    - 过期草稿自动提升/删除
    - 质量分 0 分的 Outdated 自动归档而非保留

  7. 错误处理标准化

  - MCP 返回的是 string JSON，但错误格式不统一
  - 有的返回 {"error": true, ...}，有的返回 {"message": "...", "error": true}
  - 建议：统一错误响应 envelope

  8. 批量操作支持

  - 当前工具都是单条操作
  - memory_save / memory_recall 无批量版本
  - 建议：添加 memory_batch_save、memory_batch_recall
