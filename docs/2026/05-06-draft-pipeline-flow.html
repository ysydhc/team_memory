📋 Draft 输入 → 过滤 → 处理 → 生成经验 流程图

📥 输入源
Obsidian Watcher 监听 vault 文件变更，触发 /hooks/after_response
Agent Hook tm_hook.py after-response 发送 response_text
MCP Tool memory_draft_save 直接保存草稿
💾 DraftBuffer 写入 (SQLite)
scripts/daemon/draft_buffer.py — ~/.cache/tm-pipeline/drafts.db
状态: pending → 等待 RefinementWorker 处理
INSERT INTO drafts (id, title, content, project, status='pending')
每 30 秒扫描一次
⚙️ RefinementWorker 扫描 (scripts/daemon/refinement_worker.py)
get_needs_refinement() 获取所有 status='pending' 的 draft
🔍 判断：是否学习卡片？
标题匹配：卡片-* 主题-* Layer * 学习计划* 客户端 -*
内容匹配：学习目标 前置知识 学习路径 Layer 0-4 扫盲 知识地图
✅ 是学习卡片
直接发布
mark_published(draft_id)
跳过 LLM 抽取，不提取 solution
DraftBuffer 状态 → published
❌ 不是学习卡片
🤖 LLM 精炼
调用 parse_content_openai()
模型: GLM-5-Turbo (via LiteLLM)
提取: title, problem, solution, tags
POST http://localhost:4000/v1/chat/completions
📝 TM Sink 保存 (scripts/daemon/tm_sink.py)
op_draft_save() → 写入 PostgreSQL experiences 表
状态: draft → published（如果 convergence 检测通过）
🔢 Embedding + FTS 索引
Embedding qwen3-embedding:0.6b (1024d) via Ollama
FTS jieba 分词 → to_tsvector('zhcfg', ...)
向量 存入 experiences.embedding (vector(1024))
✅ Published Experience
完整记录: title + problem + solution + tags + embedding + fts
可被 memory_recall 搜索到
可被 Faithfulness 系统评估使用率

⚠️ 已知问题

  • 学习卡片重复：同一文件多次修改会创建多条 draft（watcher 去重不够）
  • LLM 抽取失败：超时或格式错误时 fallback 发布原始内容
  • 收敛检测：需要多次修改才触发 published（convergence detector）
