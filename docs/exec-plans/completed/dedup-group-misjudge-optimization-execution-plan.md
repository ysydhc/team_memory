# 去重检测经验组误判优化 — 执行计划（可勾选任务清单）

> 完整设计与三 Agent 评审合并见 **[docs/plans/2025-03-03-dedup-group-misjudge-optimization.md](../../docs/plans/2025-03-03-dedup-group-misjudge-optimization.md)**。本文件为当前轮次的可执行任务清单，按依赖顺序执行，完成后可勾选。

---

## 背景与目标（摘要）

- **问题**：去重页仅比较根经验 embedding，经验组父向量未含子或为短模板时，两个子任务完全不同的组被误判为 100% 相似。
- **目标**：解决误判并从算法与流程上保证后续不再出现同类问题。
- **策略**：去重侧经验组感知过滤（子集明显不同则排除）+ API 返回子信息 + 合并/ reembed 风险闭环 + 回归与防回退测试。

---

## 迭代划分

- **v1（MVP）**：Task 2 + Task 3 + Task 7，辅以 Task 1 最小化；可选 Task 5。
- **v2**：Task 4（合并两组处理）+ Task 5（若未做）+ Task 6（前端展示与筛选）+ Task 1 完整文档。

## 阶段一：MVP — 根治误判与防复发

| 序号 | 任务 | 状态 | 说明 |
|------|------|------|------|
| 1.1 | **Task 1（最小化）** 设计原则与入口说明 | ⬜ | 写一条设计原则并链到 docs/plans/2025-03-03-dedup-group-misjudge-optimization.md；在 `find_duplicates` 注释注明「经验组对需结合子集过滤」；去重页「刷新经验组向量」旁加「经验组建议先执行刷新再扫描」提示。 |
| 1.2 | **Task 2** 去重 API 返回子信息 | ⬜ | `find_duplicates` 返回 `exp_a`/`exp_b` 含 `children_count`、`children_preview`；用 1 次批量 load 根+children（selectinload），避免 2N 次查询；补充测试断言返回结构。 |
| 1.3 | **Task 3** 经验组感知后过滤 | ⬜ | 双方均为经验组且子标题 Jaccard < 0.2 时从去重结果**排除**该对（仅排除、不降权）；可配置阈值；新增测试：两组父描述极似、子集明显不同 → 不在结果中。 |
| 1.4 | **Task 7** 回归与防回退测试 | ⬜ | 回归用例：两组父描述相同/极似、子集合明显不同 → 断言该对不在去重列表；父向量含子：save_group 后父 embedding 与「父+子」更近、与「仅父」更远；防回退：仅父比较时该用例必须失败。 |
| 1.5 | （可选）**Task 5** reembed 限流/分批 | ⬜ | `reembed_experience_groups` 分批（如 50～100 条/批）、批间限流；返回 `errors` 含 root_id；文档约定部分失败时重跑或人工处理。 |

## 验收标准（与 docs/plans 一致）

- 误判已解决：原「托底修复 vs 架构决策」类两组不再出现在去重高相似列表；回归用例通过。
- 后续不复发：设计原则与代码注释到位；Task 7 防回退测试通过；恢复「仅父向量比较」会导致该测试失败。
- 风险已闭环：合并两组均为父时有禁止或 reparent；reembed 有分批/限流与 errors 可重跑。
- 所有改动通过 `ruff check src/` 与相关 pytest（test_lifecycle、test_embedding_text、test_web）。
