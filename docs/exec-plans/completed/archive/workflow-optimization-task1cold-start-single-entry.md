# 第一项任务：冷启动只在一个入口（仅 step-0）

> **【已归档】** 已完成的子任务详情，仅作历史参考。
>
> 对应改进清单「一、流程与 Rules 收口」第 2 条；任务组 workflow-optimization 中第一条待办。  
> 已落地；经验已写入 TM（best_practice）：工作流冷启动只在一个入口（仅 step-0）。

---

## 一、要解决的问题

### 1.1 现象

当前流程里，「冷启动」（补全任务组描述、子任务 description、验收标准、风险点等）**有两处入口**：

- **step-0**：在认领前做——检查任务组/子任务描述是否完善，若不完善则冷启动并写回，再进入 step-1。
- **step-1**：认领后——「认领后自动检查：检查该任务状态（如 description 是否为空）；若有缺少项（未冷启动），**先执行 step-0 或冷启动补全 description**，再进入 step-2。」

### 1.2 导致的问题

- **语义两套**：冷启动既可以在「认领前」做（step-0），也可以在「认领后」做（step-1 里「或冷启动」）。Agent 和人都容易产生歧义：到底该先认领再冷启动，还是先冷启动再认领？
- **记录不清**：认领后在 step-1 里做的「冷启动」该记成 step-0 完成还是 step-1 的一部分？状态文件和步骤审计（tm_message step-X）会不一致。
- **可重复性差**：有时跳过 step-0 直接认领，再在 step-1 里补描述；有时又先跑 step-0。行为不统一，不利于「只做下一步」「从断点继续」的约定。

### 1.3 目标

- **冷启动只在一个入口**：**仅 step-0** 承担「冷启动」语义。
- step-1 认领后若发现该任务 description 为空或不足（未冷启动），**不允许**在 step-1 内再定义一套「顺带冷启动」；只能：
  - **补做 step-0**（执行 step-0 的冷启动动作，补全描述并写回），并在状态与审计中**记为 step-0 完成**，再进入 step-2；或
  - **退回 step-0**（将当前步骤视为 step-0，执行冷启动后再回到 step-1 或继续）。
- 这样「是否已冷启动」「冷启动算哪一步」唯一由 step-0 决定，便于理解和工具解析。

---

## 二、实现方式

### 2.1 原则

- 不增加新步骤，不改变 step 编号；只收口「冷启动」的语义与表述，统一到 step-0。
- 修改三处：**流程 YAML**、**设计说明**、**Rules**（可选一句收口）。

### 2.2 具体改动

#### （1）流程定义 YAML：task-execution-workflow.yaml

- **step-0**：保持现状，明确为「唯一冷启动入口」；可在注释或 name 中加一句「冷启动仅在此步」。
- **step-1 的 action**：  
  将「若有缺少项（未冷启动），先执行 step-0 或冷启动补全 description，再进入 step-2」  
  改为：  
  **「若该任务 description 为空或不足（未冷启动），须补做 step-0 的冷启动动作（补充任务组/子任务 description 并写回），用 tm_message 记录为 step-0 完成后再进入 step-2；不得在 step-1 内以『顺带冷启动』名义跳过 step-0 语义。」**

这样 step-1 不再出现「或冷启动」的第二入口，只保留「发现未冷启动 → 补做 step-0 并记为 step-0」。

#### （2）设计说明：task-execution-workflow-design.md

- 在「一、如何编辑本工作流」或「二、运行机制」中增加一条约定：  
  **「冷启动单入口」**：冷启动仅在 step-0 执行。step-1 认领后若发现任务 description 为空或不足，须补做 step-0 的冷启动动作并记为 step-0 完成（或退回 step-0），再进入 step-2；不得在 step-1 内定义另一套冷启动语义。

#### （3）Rules：.cursor/rules/tm-core.mdc

- 在「按任务执行工作流时」下增加一句：  
  **「冷启动仅在 step-0 执行；认领后若该任务 description 为空或不足，须补做 step-0 的冷启动动作并记为 step-0 完成，再进入 step-2。」**

### 2.3 验收标准

- YAML 中 step-1 的 action 已无「或冷启动」表述，且明确「须补做 step-0…并记为 step-0 完成」。
- 设计文档中有「冷启动单入口」约定。
- Rules 中有对应一句收口（可选但建议）。
- 阅读流程与 Rules 时，能唯一得出「冷启动 = step-0」的结论。

## 四、改动效果

- **语义唯一**：冷启动仅与 step-0 绑定，Agent 与人在「何时做冷启动」「记在哪一步」上无歧义。
- **状态与审计一致**：认领后补做的冷启动统一记为 step-0 完成，状态文件与 tm_message step-X 一致，便于「只做下一步」与断点恢复。
- **两套 YAML 一致**：task-execution-workflow 与 workflow-optimization-workflow 的 step-0/step-1 表述对齐，共用同一套冷启动约定。
- **可检索**：本约定已写入设计文档与 Rules，后续改流程或写新工作流时可通过文档/经验检索复用。
