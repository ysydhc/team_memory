# 经验提取与检索 — 执行指南

> **用途**：供 Agent 快速查阅执行，也供人工维护与补充。完整规则见 `.cursor/rules/tm-extraction-retrieval.mdc`。

---

## 一、一句话概括

- **提取**：当对话/计划/任务产生「对后续有 actionable 影响」的内容时，用 `tm_learn` 或 `tm_save` 沉淀。
- **检索**：新会话必调 `tm_preflight`；执行中遇到相关任务、报错、选型时，用 `tm_search` 或 `tm_solve` 获取经验。

---

## 二、提取触发检查清单

满足以下**任一**项时，应评估并执行经验提取：

| # | 条件 | 评估要点 |
|---|------|----------|
| 1 | 对话轮数 ≥ 10 | 是否包含问题—解决—验证的完整信息？ |
| 2 | 用户说「保存」「记住」「沉淀」「总结到经验库」 | 直接执行 |
| 3 | 用户说「谢谢」「再见」「先这样」「可以了」 | 会话收尾，评估是否有可沉淀内容 |
| 4 | 用户确认「解决了」「搞定」 | 问题已解决，提取解决过程 |
| 5 | 对话持续 > 30 分钟 | 长时交互，更可能包含可复用经验 |
| 6 | 涉及代码/配置/架构改动 | 用户未反对沉淀时执行 |

**原则**：若评估为「无价值」或「重复」，可不提取；宁可少写，不可滥写。

---

## 三、检索触发检查清单

满足以下**任一**项时，应主动检索经验：

| # | 条件 | 动作 |
|---|------|------|
| 1 | 新会话开始 | **必须**调用 `tm_preflight`，按 search_depth 执行 |
| 2 | 任务描述与已有经验相关 | 调用 `tm_search` 或 `tm_solve` |
| 3 | tm_preflight 返回 full | 追加 `tm_search` 或 `tm_solve` |
| 4 | 用户说「查一下经验」「参考之前的做法」 | 直接检索 |
| 5 | 遇到报错或卡住 | 在尝试修复前先检索类似问题 |
| 6 | 技术选型/架构决策 | 检索 tech_design、best_practice |
| 7 | 创建或执行 Plan | 无论 preflight 结果，**必须**全量检索 |

---

## 四、工具选择决策树

```
需要沉淀经验？
├─ 是：内容形式？
│   ├─ 长对话 / 多步骤 / 计划执行过程 → tm_learn(conversation=..., as_group=True)
│   ├─ 单点经验（问题+解法） → tm_save(title=..., problem=..., solution=...)
│   └─ 任务完成复盘 → 按 task-sediment-extract 规范 + tm_save/tm_save_typed
│
需要检索经验？
├─ 是：检索目的？
│   ├─ 具体技术问题，要最佳方案 → tm_solve
│   ├─ 探索性 / 多结果 / 工作流 → tm_search（工作流可带 tags=["workflow"]）
│   └─ 基于当前上下文推荐 → tm_suggest
│
个人偏好？
└─ 用户表达语言/习惯/偏好 → tm_save，tags 含 user_preference
```

## 五、执行示例

### 5.1 提取示例

**场景**：用户说「把这段对话总结到经验库」，对话共 12 轮，涉及一个 FastAPI 超时问题的排查与修复。

**执行**：
1. 评估：有价值 → 执行提取
2. 内容为长对话 + 多步骤 → 使用 `tm_learn`
3. 调用：`tm_learn(conversation=<对话全文>, as_group=True, tags=["fastapi", "timeout"])`

**场景**：用户说「谢谢，搞定」，此前仅 3 轮对话，修复了一个小配置问题。

**执行**：
1. 评估：轮数少但问题已解决，若配置变更有复用时 → 可提取
2. 单点经验 → 使用 `tm_save`
3. 调用：`tm_save(title="xxx 配置修复", problem="...", solution="...", tags=["config"])`

### 5.2 检索示例

**场景**：用户新开会话，说「帮我修复 Web 端的登录问题」。

**执行**：
1. 调用 `tm_preflight(task_description="修复 Web 端登录问题", current_files=[])`
2. 若返回 `search_depth: full` → 追加 `tm_search(query="Web 登录问题", tags=["web"])` 或 `tm_solve(problem="Web 端登录问题")`
3. 将检索结果注入上下文，再执行修复
