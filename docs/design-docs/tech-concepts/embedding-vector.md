# 向量与 Embedding

> 技术概念 | 不涉及项目实现
> 相关：[pgvector-fts](pgvector-fts.md) | [reranker](reranker.md)

## 什么是向量嵌入 (Embedding)？

**通俗解释**：把一段文本变成一串数字（向量），就像在 768 维空间里给它定一个坐标。语义相近的文本，坐标会靠得更近。

**为什么有用**：
- 传统搜索靠关键词匹配："Docker 网络不通" 只能找到包含这几个字的记录。
- 向量搜索靠语义："容器之间 ping 不通" 和 "Docker 网络不通" 含义接近，向量距离近，能互相被搜到。

**示例**：
```
"Docker 容器无法启动"  →  [0.12, -0.34, 0.56, 0.78, ...]  (768 个数字)
"Docker container fails to start"  →  [0.11, -0.33, 0.57, 0.79, ...]  (768 个数字)
"今天天气真好"  →  [0.89, 0.22, -0.67, 0.01, ...]  (768 个数字)
```

- 前两个向量非常接近（数字很相似），因为**含义相近**
- 第三个向量和前两个差别很大，因为**含义完全不同**

这些向量由 **嵌入模型**（Embedding Model）计算，模型读了海量文本，学会了把"含义"编码成数字。

## 什么是余弦相似度？

有了向量，怎么衡量两段文字有多相似？

**余弦相似度** 衡量两个向量的"方向"是否一致（忽略长度）：

| 值 | 含义 |
|----|------|
| 1.0 | 完全相同方向（含义一致） |
| 0.0 | 完全正交（毫无关系） |
| -1.0 | 完全相反方向（含义相反） |

**余弦距离** = 1 - 余弦相似度。距离越小，越相似。

简单的数学例子（2 维示意）：
```
A = [1, 0]    B = [1, 0]    → 余弦相似度 = 1.0（完全一样）
A = [1, 0]    B = [0, 1]    → 余弦相似度 = 0.0（完全无关）
A = [1, 0]    B = [0.9, 0.1] → 余弦相似度 ≈ 0.99（非常相似）
```
