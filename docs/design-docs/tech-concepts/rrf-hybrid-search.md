# RRF 与混合检索

> 技术概念 | 不涉及项目实现
> 相关：[embedding-vector](embedding-vector.md) | [pgvector-fts](pgvector-fts.md)

## 为什么需要混合检索？

单一搜索方式各有盲区：
- **向量搜索的盲区**：可能返回语义相近但实际不相关的结果
- **FTS 的盲区**：无法处理同义词和不同表述

混合检索同时使用两种方式，然后合并结果，取长补短。

## RRF（Reciprocal Rank Fusion）原理

RRF 是一种简单但有效的分数融合算法。它不关心各个搜索方式给出的具体分数（因为不同搜索方式的分数不可比），而是关心**排名**。

**公式**：
```
RRF_score(d) = Σ (weight / (rank + k))
```

其中：
- `d` 是某个文档
- `rank` 是该文档在某个搜索结果中的排名（从 0 开始）
- `k` 是常数（默认 60，经过学术研究证明的最优值）
- `weight` 是该搜索方式的权重

**示例**：查询 "Docker 网络问题"

向量搜索结果：A(0), B(1), C(2)
FTS 搜索结果：B(0), D(1), A(2)

权重：vector=0.7, fts=0.3

- 文档 A：0.7/(0+60) + 0.3/(2+60) = 0.01167 + 0.00484 = **0.01651**
- 文档 B：0.7/(1+60) + 0.3/(0+60) = 0.01148 + 0.00500 = **0.01648**
- 文档 C：0.7/(2+60) = **0.01129**
- 文档 D：0.3/(1+60) = **0.00492**

最终排序：A > B > C > D。文档 A 和 B 都出现在两个搜索结果中，所以得分最高。
