# Reranker（精排）

> 技术概念 | 不涉及项目实现
> 相关：[embedding-vector](embedding-vector.md) | [rrf-hybrid-search](rrf-hybrid-search.md)

## 什么是 Reranker？

**生活化比喻**：在图书馆找书
- **第一步（粗排/召回）**：管理员根据关键词快速取出 50 本"可能相关"的书。这一步很快但不够精确。
- **第二步（精排/Reranking）**：逐一翻阅这 50 本书的目录和简介，根据具体需求重新排序，把最相关的 10 本排到前面。

在搜索系统中：
- **粗排（Retrieval）** = 向量搜索 + 全文搜索，快速召回候选结果
- **精排（Reranking）** = 用更精确的方法对候选结果重新打分排序

## 为什么需要 Reranker？

向量搜索有一个根本性局限：

- **Bi-encoder（双编码器）**：把查询和文档分别编码成向量，然后计算余弦相似度。优点是速度快（可预计算文档向量），缺点是查询和文档是"各看各的"，无法做深层交互理解。

- **Cross-encoder（交叉编码器）**：把查询和文档拼在一起输入模型，模型可以同时看到两者，做更精细的比较。精度更高但速度慢（每对都要重新计算）。

**最佳实践**：用 Bi-encoder 快速召回大量候选，再用 Cross-encoder（或 LLM）精排少量候选。

```
Bi-encoder（快但粗）          Cross-encoder/LLM（慢但精）
   ↓ 召回 50 条                    ↓ 精排 50 → 10 条
   向量搜索                         逐一打分
```
